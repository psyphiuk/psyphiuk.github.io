Quick Wins

This section includes details of some of the quick wins I had while making this site.

  • The main structure for the scss files has been easier than I expected to settle on. I have taken inspiration from 7-1 Sass Architecture but used helper instead of abstracts and don't have a vendors folder as I don't intend to use any 3rd party packages
  • Added some styling for code snippets using a monospace google font.
  • I have added a simple, at the moment, colour theme using a primary colour picked from one of the images I have downloaded from unsplashed and the sass:color built-in package.

Gotchas

This section includes things that have not gone how I expected.

  • I am using a two column aside and article structure for this page and had intended to use has to determine if an aside is present and where it is and then style appropriately. But it didn't work on my chrome browser and when I checked is not well supported yet. I have used a media query to do this for now but at some point need to explore other ways to do it.
    I have settled on a simple display: grid; for this atm.
  • I wanted to improve readability by limiting the width of a paragraph to 60ch so that it flows into 2 columns on larger screens. This didn't happen in the nice clear way I was expecting. Looking at the example on w3 schools the column-width is added to a div and the text just included directly without a p tag. I would like to format it into paragraphs so will need to come up with a solution to this.
    Extra Gotcha: Is there are other tags too. This should apply to list items and the blockquotes.
    Breakthrough: Not sure what I had done but it is working the way I wanted it to now. It also helps that each section has more text now.
  • Sticky on the aside, add top with a padding variable and the section list stays on the screen.
    Interesting point: The sticky doesn't work on mobile view. It took a while to understand why and how to fix it.
